# AI 商家询盘助手 - 未来开发规划 (截至当前)

**项目状态：** Prototype 1 的核心后端功能已完成，前端聊天界面已能与后端成功对接，RAG (检索增强生成) 流程已验证通过。

**核心原则：** 严格区分客户界面 (Chat) 与商家管理界面 (Admin)，确保数据安全与操作便捷。

---

### **第一阶段：完善前端用户体验 (Frontend UX Enhancement)**

**目标：** 将当前的聊天界面从“可用”提升到“好用”，优化视觉和交互细节。

1.  **优化聊天气泡样式**：
    *   为 AI 和用户的消息气泡添加更明确的区分（如头像、颜色、箭头等）。
    *   对代码块、列表、链接等特殊格式的 Markdown 内容进行美化渲染。
2.  **增加加载与错误状态的友好提示**：
    *   优化“正在输入...”的动画效果。
    *   为 API 请求失败、网络错误等情况设计更友好的错误提示信息。
3.  **支持流式响应 (Streaming)**：
    *   **技术挑战**：改造后端 `/api/chat` 接口，使用 `StreamingResponse`。
    *   **用户体验**：让 AI 的回复像 ChatGPT 一样逐字或逐句地“流式”出现，而不是等待全部生成完毕后一次性显示，极大地提升响应速度和用户感知。

---

### **第二阶段：构建商家后台管理中心 (Admin Panel)**

**目标：** 创建一个功能强大的后台管理页面，让商家可以轻松地管理知识库和查看客户线索。

1.  **搭建前端路由**：
    *   安装 `react-router-dom` 库。
    *   创建两个主页面路由：
        *   `/`: 指向 `Chat.tsx` (客户聊天界面)。
        *   `/admin`: 指向 `Admin.tsx` (商家后台管理界面)。
2.  **开发后台管理页面 `Admin.tsx`**：
    *   使用 Tab 或侧边栏导航，区分“知识库管理”和“客户线索”两大功能区。
3.  **实现知识库管理模块**：
    *   **前端**：
        *   开发一个文件上传组件，支持拖拽或点击上传 `.pdf`, `.docx`, `.txt` 文件。
        *   开发一个知识列表，用于展示当前所有的知识文档，并提供“删除”功能。
    *   **后端 (API)**：
        *   创建新的 `/api/admin/upload-file` 接口，用于接收上传的文件。
        *   创建新的 `/api/admin/documents` 接口，用于获取和删除知识文档。
    *   **后端 (Service)**：
        *   创建 `file_processing_service.py`，用于：
            *   **文件解析**：根据文件类型 (pdf, docx) 调用相应的库 (如 `PyPDF2`, `python-docx`) 提取纯文本。
            *   **文本切块 (Chunking)**：将长文本智能地切分成包含上下文的段落。
        *   改造 `knowledge_service.py`，使其能够接收这些文本片段，并完成后续的向量化和存储。
4.  **实现客户线索查看模块 (Lead Management)**：
    *   **前端**：
        *   开发一个客户线索列表，展示从 `lead` 表中获取的数据。
        *   开发一个聊天记录详情视图，当点击某个客户时，可以获取并展示该客户的完整对话历史。
    *   **后端 (API)**：
        *   (已存在) `/api/admin/leads`: 获取所有客户线索。
        *   (已存在) `/api/admin/conversations/{id}`: 获取指定线索的聊天记录。

---

### **第三阶段：AI 能力进阶 (AI Capability Upgrade)**

**目标：** 进一步提升 AI 的智能水平和功能边界。

1.  **实现主动引导与信息收集**：
    *   **技术挑战**：设计更复杂的 Prompt，让 AI 不仅能回答问题，还能在对话中主动识别高意向客户，并引导他们提供姓名、电话、邮箱等关键信息。
    *   **后端**：改造 `/api/chat` 接口，使其能够识别人名、电话、邮箱等实体，并在识别后自动将这些信息存入 `lead` 表。
2.  **优化检索算法**：
    *   尝试更先进的检索策略，如 `HyDE` (Hypothetical Document Embeddings) 或重排 (Re-ranking) 模型，以提升知识检索的准确率。
3.  **引入对话历史记忆**：
    *   改造 `/api/chat` 接口，使其能够接收并理解之前的对话历史，实现真正的多轮对话，而不是现在这样每次都只看当前问题。
