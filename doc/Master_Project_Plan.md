## 1. 项目概览 (Project Overview)

### 1.1 项目目标
开发一个面向中小商家（如诊所、装修团队、健身房等）的“AI 原型系统”，旨在将普通的“客户咨询”转化为“可跟进的销售线索（Lead）”。此阶段为 Prototype 1 原型，**核心验证点**为：
- LLM 是否能依据知识库准确回答顾客常见问题（不瞎编）。
- 系统是否能在自然对话中自动收集客户的核心联系方式（如姓名、电话）。
- 商家是否能通过后台管理页高效查看意向客户并进行跟进。

### 1.2 核心系统流程测试场景 (Demo Scenarios)
- **场景 A (基础问答):** 用户咨询服务项目/营业时间等，系统精准检索知识库并返回答案。
- **场景 B (高意向引流):** 用户询问确切价格或表达预约意向，系统除了回答问题外，必须**顺势引导用户留下手机号或邮箱**。
- **场景 C (后台校验):** 商家登录管理端查看刚产生的新线索，包含被自动判断出的“意向等级（高）”和“一句话摘要”。

## 2. 技术架构与原理解释 (Architecture & Tech Stack)

系统分为**前台大堂（Frontend）**和**幕后厨房（Backend & AI）**两套独立工程。

### 2.1 Backend 后台系统 (数据处理与核心 API)
- **语言框架:** Python + FastAPI 
- **数据库:** MySQL (作为核心账本持久化保存数据)
- **ORM 数据操作:** SQLModel (Python 数据库交互工具)
- **结构划分:**
  - `models/`: 数据库表结构定义。
  - `api/`: 对外提供的收发消息与管理接口。
  - `services/`: 业务逻辑，分为对话链路 (`llm_service`)和检索链路 (`rag_service`)。

### 2.2 Frontend 前端界面 (客户展示与后台管理)
- **开发栈:** React.js (Vite Vite) + TypeScript + Tailwind CSS
- **核心页面:**
  - 响应式聊天页面：支持移动桌面双端，包含输入框和聊天气泡流。
  - Lead 后台管理页：展示客户列表、电话追踪与历史聊天记录详情。

### 2.3 AI 向量大脑 (向量检索与 LLM)
- 大模型大脑依靠第三方 **OpenAI-Compatible API**（如 DeepSeek、Kimi 等，需配置 API Key）。
- 检索知识库依靠**轻量级本地向量组件**（初期推荐使用 FAISS 或 ChromaDB），只加载商家专属 FAQ 文本转换为向量密码与用户提问做比对。

## 3. API 开发与数据模型 (API & Data Models)

### 3.1 核心数据表 (MySQL)
1. **Lead 表:** 存潜客信息 (`id`, `name`, `phone`, `email`, `intent_level`, `summary`)
2. **Conversation 表:** 存会话大类 (`id`, `lead_id`, `session_id`)
3. **Message 表:** 存具体消息历史 (`id`, `conversation_id`, `role`, `content`)
4. **KnowledgeDocument/Chunk 表:** 存被切块的 FAQ 信息库以供检索。

### 3.2 核心接口设计 (RESTful API)
- `POST /api/chat/message`: 接收用户输入，返回 AI 回复并检测留资。
- `GET /api/admin/leads`: 后台获取所有线索列表。
- `GET /api/admin/conversations/{id}`: 后台获取该线索下的详细聊天记录。
